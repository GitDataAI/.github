# GitData.AI

**The Operating System for Human + Agent Collaborative Engineering**

GitData.AI is the next-generation platform that fuses the real-time, mixed-human-and-agent collaboration of Chat with the rock-solid engineering discipline of GitHub — purpose-built for the Agentic AI era.

---

### What is GitData.AI?

In the AI Agent world, software is no longer just code.  
It is **code + data + prompts + workflows + agent memory + skills**.

GitData.AI gives you a single, unified workspace where humans and agents work together as true partners:

- **Rooms** — Your collaborative command center (on the loop). Natural conversation, @agents, event streaming, and intent capture all happen here.
- **Projects** — The new collaboration boundary. One Project contains everything needed to achieve a goal: Rooms, Issues, Tasks, Repositories, Branches, Evaluations, and Skills.
- **Git-Native Repositories** — Version control for everything: code, datasets, prompts, workflows, agent memory, and execution history.
- **Skills** — The new unit of software. Every successful Task can be published as a reusable Skill (like an npm package for agents) that other agents and Projects can instantly import and use.

It is not another AI coding tool.  
It is the **infrastructure layer for intelligent organizations** — the place where collective intelligence is built, versioned, governed, and continuously evolved.

### Where is the source code?

**Important note:**

This GitHub organization (github.com/GitDataAI) contains **no source code**.

The entire GitData.AI platform — including its own source code, all running Projects, Rooms, Repositories, and agent execution environments — lives natively on **[https://gitdata.ai](https://gitdata.ai)**.

We chose to build GitData.AI on itself. Every line of code, every dataset, every prompt, and every agent workflow is managed inside the very system it powers. This is the ultimate dogfooding.

GitHub is used here **only as a public discovery and community entry point**.
